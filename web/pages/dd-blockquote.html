Blockquote Component Breakdown

Overview

The dd-blockquote component provides a styled blockquote for testimonials, quotes, or highlighted statements. It includes optional elements like an icon, person's image, name, title, and JSON-LD schema for SEO.

This component is ideal for testimonials sections, blog pull-quotes, or about pages. In WordPress, configure it via ACF fields for quote text, name, title, image, and schema. In Drupal, use a Paragraph type with text fields for quote and details, plus an image field, rendering schema dynamically.

Basic Usage

Use a blockquote with class dd-blockquote and data-aos for animations. Include structured elements for icon, person details, and comment. Add schema script for structured data.

        , Title
      

      

        
Excepteur sint occaecat cupidatat non proident, sunt in culpa qui officia deserunt mollit anim id est laborum.

Repeat for multiple quotes in a loop, using grid for layout.

CSS Modifiers (from _dd_blockquote.scss)

The SCSS file defines base styles for the blockquote, including centering, icons, and responsive sizing. Key modifiers include:

  • -left: Aligns content to the left.
  • -right: Aligns content to the right.
  • -no-icon: Removes the quote icon.
  • -no-image: Hides the person's image for text-only quotes.
  • -compact: Reduces spacing and sizes for inline or smaller displays.
  • Utility classes for alignment. -x-center -y-center
  • AOS data attributes for custom animations. data-aos="fade-up"

Framework files

  • source/js/vendors/aos.js (animations)
  • source/scss/components/_dd_blockquote.scss (styles)

For ACF/Paragraphs integration, map fields to generate schema and classes dynamically.

Accessibility and Best Practices

Use alt text for images. Ensure schema is accurate for SEO. Test readability and contrast. In CMS, require name and quote fields, and validate image uploads for accessibility.
