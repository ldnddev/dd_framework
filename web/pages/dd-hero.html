Hero Component Breakdown

Overview

The dd-hero component provides a prominent introductory section, typically at the top of a page, featuring a large background image with overlaid content like titles, body text, and calls-to-action. It's designed for engaging headers with responsive backgrounds and flexible content alignment.

This component is ideal for homepages, landing pages, or section intros. In WordPress, configure it via ACF fields for hero image (with srcset), title, body, links, and aria-label. In Drupal, use a Paragraph type with media field for image, text fields for content, and optional link fields, rendering responsive attributes dynamically.

Basic Usage

Use a section with class dd-hero and aria-label for accessibility. Include an image with srcset for responsiveness, and content with title, body, and optional links.

CSS Modifiers (from _dd_hero.scss)

The SCSS file defines base styles for the hero, including absolute positioning for images on desktop, flexible content alignment, and responsive heights. Key modifiers include:

  • -left: Aligns content to the left.
  • -center: Centers content (default for mobile).
  • -right: Aligns content to the right.
  • -overlay-light: Adds a semi-transparent overlay for better text readability on images.
  • -overlay-dark: Adds a darker semi-transparent overlay for better text readability on images.
  • Utility classes for alignment. -x-center -y-center
  • AOS data attributes for custom animations. data-aos="fade-up"

For ACF/Paragraphs integration, map image fields to generate srcset and classes dynamically.

Accessibility and Best Practices

Use aria-label for the section. Provide descriptive alt text for images. Ensure text contrasts with backgrounds. Test keyboard focus on links. In CMS, enforce alt text and aria requirements, and validate for mobile stacking order.
