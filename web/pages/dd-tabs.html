Tabs Component Breakdown

Overview

The dd-tabs component provides a tabbed interface for organizing content into switchable panels. It supports desktop navigation and a mobile dropdown menu, with ARIA attributes for accessibility, making it suitable for FAQs, product details, or multi-section content without page reloads.

This component is ideal for content-heavy pages where space-saving is key. In WordPress, configure it via ACF repeater fields for tab items (e.g., label, content, links). In Drupal, use a Paragraph type with fields for tab titles and bodies, rendered with unique data-id for scoping multiple instances.

Basic Usage

Wrap tabs in a div with class dd-tabs and a unique data-id. Include navigation with menu items (ARIA roles) and content panels with aria-hidden and role="tabpanel".

CSS Modifiers (from _dd_tabs.scss)

The SCSS file defines base styles for tabs, including responsive navigation (hidden menu on mobile), active states, and overflow handling. Key modifiers include:

  • -active: Applied to active menu items and panels for visibility and styling.
  • -horizontal: Adjusts for horizontal tab layouts if extended.
  • -vertical: For vertical tab orientations (sidebar style).
  • Utility classes like AOS data attributes (e.g., data-aos="fade-in") for panel animations.

Associated JavaScript (from dd_tabs.js)

The component includes a JS file handling mobile menu toggle and tab switching with ARIA updates. Key features:

  • Toggles -active class on mobile menu open/close.
  • Switches tabs on click, updating aria-selected, aria-hidden, and active classes, scoped by aria-controls.
  • Updates mobile placeholder text to current tab label.
  • Supports multiple tab groups on a page via aria-controls and panel ID.

Include the JS via your CMS: In WordPress, enqueue in functions.php; in Drupal, attach via libraries.yml.

For ACF/Paragraphs integration, generate unique aria-controls from fields.

Accessibility and Best Practices

Use ARIA roles (tablist, tab, tabpanel) and attributes (aria-selected, aria-controls, aria-labelledby, aria-hidden). Ensure keyboard navigation (focus on tabs, enter/space to activate). Test with screen readers for announcements. In CMS, require unique labels and validate content for empty panels.
