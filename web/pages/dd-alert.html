Alert Component Breakdown

Overview

The dd-alert component provides a flexible way to display notification messages, such as warnings, errors, successes, or informational alerts. It supports different severity levels with color-coded visuals for quick user recognition.

This component is ideal for form validation feedback, system messages, or user notifications. In WordPress, configure it via ACF fields for alert type, heading, and text. In Drupal, use a Paragraph type with select fields for type and text areas for content, allowing dynamic rendering based on conditions.

Basic Usage

Wrap the alert content in a container with class dd-alert and role="alert" for accessibility. Include a heading and text, with optional links. Use AOS for fade-in animations.

CSS Modifiers (from _dd_alert.scss)

The SCSS file defines base styles for alerts, including colors, borders, and icons based on type. Key modifiers include:

  • -info: Blue-themed for informational messages.
  • -success: Green-themed for positive confirmations.
  • -warning: Yellow/orange-themed for cautions.
  • -error: Red-themed for errors or critical alerts.
  • -compact: Reduces padding for inline or space-constrained uses.
  • Utility classes for alignment. -x-center -y-center
  • AOS data attributes for custom animations. data-aos="fade-up"

Accessibility and Best Practices

Use role="alert" for immediate announcements. Ensure color contrast meets WCAG standards. Include icons for visual cues but hide them from screen readers if decorative. Test with screen readers for live updates. In CMS, make text fields support HTML for links, and validate for empty content.
