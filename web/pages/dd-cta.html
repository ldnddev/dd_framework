CTA Component Breakdown

Overview

The dd-cta component provides a prominent call-to-action section with a background image or video, overlaid content (title, body, links), and flexible positioning for the copy block. It's designed for conversions like sign-ups, downloads, or contact forms, with responsive overlays for readability.

This component is ideal for hero CTAs, mid-page promotions, or footers. In WordPress, configure it via ACF fields for background image, title, copy, links, and position select. In Drupal, use a Paragraph type with media field for background, text fields for content, and multi-value fields for links, rendering positioning via a select widget.

Basic Usage

Wrap the CTA in a div with class dd-cta. Include an image for background (hidden on desktop for CSS background), content overlay with title, body, and links. Use grid for link alignment.

CSS Modifiers (from _dd_cta.scss)

The SCSS file defines base styles for the CTA, including cover backgrounds and content positioning. Key modifiers for .dd-cta__content include:

  • -bg-top-left: Positions copy at top-left of background.
  • -bg-top-center: Positions at top-center.
  • -bg-top-right: Positions at top-right.
  • -bg-center-left: Positions at center-left.
  • -bg-center-center: Positions at center-center (default).
  • -bg-center-right: Positions at center-right.
  • -bg-bottom-left: Positions at bottom-left.
  • -bg-bottom-center: Positions at bottom-center.
  • -bg-bottom-right: Positions at bottom-right.
  • Utility classes for alignment. -x-center -y-center
  • AOS data attributes for custom animations. data-aos="fade-up"

For ACF/Paragraphs integration, map fields to generate inline styles and classes dynamically.

Accessibility and Best Practices

Ensure overlay text contrasts with background (use borders/opacity). Provide alt text for images if visible. Use semantic headings for titles. Test focus on links. In CMS, validate background images for required fields and enforce contrast checks if possible.
