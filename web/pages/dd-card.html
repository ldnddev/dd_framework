Card Component Breakdown

Overview

The dd-card component provides a flexible card layout for displaying grouped content, such as features, products, or team members. It supports grids of cards with images, titles, subtitles, body text, and links, making it responsive and customizable.

This component is ideal for galleries, portfolios, or feature lists. In WordPress, configure it via ACF repeater fields for card items (e.g., image, title, subtitle, copy, link). In Drupal, use a Paragraph type with fields for each card element, rendered in a loop for multi-card displays.

Basic Usage

Wrap cards in a container with class dd-card. Use a grid for items, each with content structure. Add AOS for fade animations and lazy loading for images.

CSS Modifiers (from _dd_card.scss)

The SCSS file defines base styles for cards, including borders, shadows, and responsive grids. Key modifiers include:

  • -horizontal: Lays out card content horizontally (image left, copy right) instead of vertical.
  • -no-image: Hides the image for text-only cards.
  • Utility classes for alignment. -x-center -y-center
  • AOS data attributes for custom animations. data-aos="fade-up"

For ACF/Paragraphs integration, use repeater fields to generate cards dynamically, with optional fields for modifiers.

Accessibility and Best Practices

Use alt text for images and semantic headings in titles. Ensure links are descriptive. Add role="list" to items for screen readers if in a grid. Test keyboard navigation and focus styles. In CMS, require alt text and validate content for completeness.
